# Team Management

The Team Management page lets workspace admins manage members, roles, and access permissions.

<!-- IMAGE: Team Management page showing member list with roles -->

{% hint style="info" %}
Team Management is only visible to workspace admins.
{% endhint %}

## Member list

The main view shows all workspace members:

| Column | Description |
| --- | --- |
| **Name** | Member's display name |
| **Email** | Account email address |
| **Role** | Admin, Member, or Viewer |
| **Joined** | Date they joined the workspace |
| **Last active** | Most recent activity |
| **Status** | Active or Invited |

## Quick actions

- **+ Invite Member** — send an invitation to join the workspace
- **Change Role** — promote or demote a member
- **Remove** — revoke workspace access

## Workspace isolation

Each workspace is fully isolated:
- Members only see agents, documents, and data within their workspace
- Agents from one workspace cannot access another workspace's data
- Settings and credentials are workspace-scoped
